# EeveeAgent

#### 介绍
从0到1开发智能Agent


业务模块标准结构：
├── user/        # 用户模块（示例）
│   │   ├── controller/  # 接口层：只接收请求、返回数据
│   │   ├── service/     # 业务层：核心逻辑（接口+impl）
│   │   ├── mapper/      # 数据层：操作数据库
│   │   ├── domain/      # 数据库实体（PO）：和表一一对应
│   │   ├── dto/         # 数据传输对象（解耦核心！）
│   │   │   ├── req/     # 前端→后端 请求参数（如UserLoginReq）
│   │   │   ├── vo/      # 后端→前端 响应数据（如UserInfoVO）
│   │   ├── enums/       # 模块专属枚举（用户状态、角色）

开发常用数据对象分类
DTO：接收前端传过来的参数 前端传给后端
DO：和数据库表一一对应（操作数据库用）
VO：返回数据给前端展示，后端传给前端

前端页面
